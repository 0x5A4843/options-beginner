# 页面体验要求

本文档记录期权入门教程页面已经确认的布局、交互和视觉规则。后续新增屏幕或修改页面时，优先遵守这里的约束。

## 项目定位

- 主要面向已经交易期货、但对期权不熟的新手。
- 首要目标是吸引客户对期权产生兴趣，讲解知识点放在第二位。
- 内容应口语化、具体、有场景感，避免学术化和大段解释。
- 背景设定只用于指导对话设计，不直接显示在页面上。

## 页面结构

- 页面以手机应用界面为主，不做桌面优先布局。
- 第一屏直接进入教程体验，不做营销落地页。
- 每一屏由四个固定区域组成：
  - 顶部应用标题区。
  - 当前屏标题区。
  - 可选的图表或视觉区。
  - 固定高度的对话区和底部按钮。
- 页面整体不能因为对话增加而变长，必须保持在手机视口内。
- 图表出现后应保持位置稳定，不能因为新对话出现而上下跳动。

## 标题规则

- 屏幕标题要短，尽量保持一行。
- 标题不宜过大，避免占用对话和图表空间。
- 标题过长时允许单行省略，不允许把顶部撑高。
- 标题文案应描述当前屏的交易问题或能力点，例如“不判断方向，押注大波动”。
- 不要在标题里过早暴露策略名，除非用户已经在对话中确认该策略名应出现。

## 对话规则

- 固定使用两个角色：
  - `期权新手`：提出期货交易者视角的问题。
  - `期权老师`：用口语化方式回答。
- 老师和学生头像要明显不同。
- 不同角色的头像和文字应分布在对话框两侧，形成清晰的一问一答。
- 点击按钮一次，只新增一个人的一个完整对话框。
- 不要把同一个人的每句话都拆成单独点击。
- 同一个人一段话可以在一个气泡里拆成多行短句。
- 如果同一个人说得太长，再拆成两个对话框。
- 对话区高度固定，新增对话时在对话区内部滚动。
- 点击下一段时，对话滚动应从当前滚动位置继续，不要每次从顶部重新滑到最新内容。
- 自动滚动速度要偏慢，避免过快跳动。

## 按钮规则

- 对话未结束时，按钮使用类似“下一段”的动作文案。
- 当前屏全部对话显示完后，按钮文案切换为下一屏动作，例如“看一个例子”“看下一个例子”。
- 按钮固定在底部，不随对话内容移动。

## 图表规则

- 图表应服务于当前对话，不要一开始把所有信息都露出。
- 图表信息应随着对话逐步出现。
- 第二屏当前规则：
  - 图表本质是价格图，不是先展示期权盈亏图。
  - 初始状态只展示指数横盘走势和当前价格。
  - 后续才显示 `4100 认购`、`3900 认沽` 等位置。
  - 再后续显示 `4180 以上赚钱`、`3820 以下赚钱` 和中间亏权利金区间。
- 图表应保持在同一位置，后续变化只更新图内元素。
- 图表里的文字要简短，适合手机屏幕阅读。

## 内容节奏

- 每屏只讲一个核心能力点或一个经典例子。
- 不要一上来大段解释，应一步步展开。
- 先用具体交易场景引入，再解释期权能做什么。
- 策略名不要一开始就抛出，可以在最后由老师轻轻点出。
- 例如第二屏先讲：
  - 市场已经横盘很久。
  - 认为市场会有大波动。
  - 但不能确定方向。
  - 再引出不判断方向、押注大波动的期权组合。
